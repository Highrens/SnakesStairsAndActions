# Snakes, Stairs and Actions

Простое и наглядное веб-приложение на Next.js, которое генерирует случайные игровые события для настольной игры в стиле "Змеи и лестницы".

## 🎮 Как работает игра

1. Нажимаете кнопку **«Действие!»**.
2. Система выбирает одно из событий и показывает его описание.
3. В зависимости от типа события игроку предлагаются варианты действий или происходит событие автоматически.
4. При выборе варианта отображается случайный исход и применяется эффект:
   - изменение здоровья (`health`)
   - изменение золота (`gold`)
   - перемещение по клеткам (`moveBy`, `moveTo`)
   - получение проклятий (`curses`)

## ⚙️ Основная функциональность

- Генерация случайного события по нажатию кнопки.
- Отображение списка доступных вариантов действия.
- Обработка событий с броском кубика (`rollCheck`).
- Подтверждение покупок и других действий через диалоговые окна.
- Режим исключения уже использованных событий.
- Поддержка событий без выбора — `noControl`.

## 📁 Структура проекта

- `src/app` — приложение, страница и глобальные стили.
- `src/components` — UI-компоненты и основной игровой блок `ActionButton`.
- `src/constants/actions/` — разбитые по файлам категории игровых событий.
  - `simpleActions.js` — простые события без проверок.
  - `rollCheckActions.js` — события с бросками кубиков.
  - `noControlActions.js` — события, которые происходят автоматически.
  - `index.js` — центральный экспорт общего объекта `actions2`.
- `public` — статические ресурсы: изображения, фон и иконки.

## 🚀 Как запустить локально



## ✏️ Как добавлять новые действия

1. Перейдите в папку `src/constants/actions/`.
2. Найдите нужную категорию:
   - `simpleActions.js` — если действие не содержит проверки и подтверждений.
   - `rollCheckActions.js` — если действие требует броска кубика (`rollCheck`).
   - `noControlActions.js` — если событие должно происходить без выбора игрока.
3. Добавьте новый объект события в массив `actions`.
4. Используйте уникальный `id` для каждого события.
5. Убедитесь, что каждое действие содержит:
   - `image` — путь до картинки события;
   - `description` — описание события;
   - `options` — массив вариантов или одного пассивного варианта;
   - `effects` — список эффектов в исходе.

### Пример нового события

```js
{
  id: 61,
  category: 'noControl',
  image: '/images/thief.png',
  description: 'Невидимый вор прошёл мимо.',
  options: [
    {
      name: 'Событие происходит',
      outcomes: [
        {
          text: 'Ты не заметил, как потерял 1 золото.',
          effects: [{ type: 'modifyStat', stat: 'gold', value: -1 }]
        }
      ]
    }
  ]
}
```

## 💡 Полезные советы

- Для новых событий используйте уникальные изображения из папки `public/images`.
- Если добавляете событие с покупкой или требующим проверки, не забудьте описать `confirmation` или `rollCheck`.
- После изменения `src/constants/actions/index.js` приложение автоматически будет подгружать все категории.
